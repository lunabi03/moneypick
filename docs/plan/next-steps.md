# 다음 단계 실행 계획

## 1. 리서치 & 데이터 확보 (Week 1~2)

- **데이터 소스 매핑**: 공공데이터포털, 지자체 지원금 포털, 고용노동부, 통계청, 민간 제휴 플랫폼 리스트업
- **경쟁 서비스 분석**: 토스, 핀다, 오늘의집 리워드, 프리랜서 코리아 등과 비교하여 차별 포인트 도출
- **유저 인터뷰**: 타깃 세그먼트(사회초년생, 자영업자, 프리랜서, 주부)별 3~5명 인터뷰
- **법·제도 검토**: 금융/데이터 활용 관련 규제 체크, 개인정보 수집 동의 플로우 논의

## 2. 정보 구조(IA) & 와이어프레임 (Week 3~4)

- **카테고리 구조 확정**: 5개 메인 카테고리 내 서브 분류 정의 및 네이밍
- **유저 시나리오**: 지원금 찾기, 부업 신청, 투자 정보 비교, 자격증 준비 로드맵, 할인 쿠폰 적용 등 5대 핵심 플로우 작성
- **와이어프레임 제작**: Figma 기반 모바일 우선, 데스크톱 대응 레이아웃 병행
- **UI 패턴 정의**: 카드, 리스트, 배지, 필터, 모달, 알림 등 주요 컴포넌트의 기본/호버/비활성 상태 정의

## 3. 디자인 시스템 & 프로토타입 (Week 5~6)

- **디자인 토큰 정리**: 컬러, 타이포그래피, 간격, 그림자, 아이콘 스타일 명세
- **컴포넌트 라이브러리**: Figma 라이브러리 구성, 상태 컴포넌트(CTA, 배너, 알림) 완성
- **프로토타입 제작**: 주요 플로우를 연결한 인터랙티브 프로토타입(모바일/데스크톱)
- **1차 사용자 테스트**: 사용성 테스트(5명 내외), 핵심 과제 성공률 체크, 개선안 도출

## 4. 개발 환경 구축 & API 설계 (Week 7~8)

- **프론트엔드 세팅**: Next.js, TypeScript, Tailwind, Zustand/Redux Toolkit 기준 구조 생성
- **백엔드 아키텍처 결정**: NestJS 기반 REST/GraphQL, 인증/권한 관리 설계
- **데이터 모델링**: 정보 카드, 사용자 프로필, 알림, 즐겨찾기, 로그 트래킹 스키마 설계
- **외부 연동**: 공공 API, 제휴사 API 연계 PoC, 데이터 갱신 스케줄 수립

## 5. 기능 개발 (Week 9~12)

- **MVP 범위**
  - 대시보드: 하이라이트 카드, 마감 임박, 즐겨찾기
  - 카테고리 리스트 & 상세 페이지
  - 검색 & 필터 모듈
  - 사용자 프로필/설정
  - 알림(이메일/푸시) 기본 기능
- **프론트엔드 개발**: 디자인 시스템 컴포넌트 → 페이지 구성 → 상태관리 연동
- **백엔드 개발**: 인증/인가, 데이터 수집/캐시, 알림 스케줄러, 로그/분석
- **CI/CD 구성**: GitHub Actions, Vercel 배포, staging/production 분리

## 6. 테스트 & QA (Week 13~14)

- **테스트 계획**: 단위 테스트, 통합 테스트, E2E 테스트 시나리오 작성
- **QA 수행**: 성능 측정(Lighthouse), 접근성 체크(WCAG), 브라우저/디바이스 호환성
- **보안 점검**: OWASP Top 10 체크리스트, 개인정보 취급 프로세스 검토
- **베타 사용자 모집 & 피드백 수집**

## 7. 출시 & 운영 (Week 15~16)

- **런칭 캠페인**: 마케팅 커뮤니케이션 플랜, 파트너십 발표
- **운영 지표 모니터링**: 월간 활성 사용자, 전환률, 알림 반응률, 문의/오류 레이트
- **지속 개선 로드맵**: 데이터 자동화, 추천 알고리즘 고도화, 커뮤니티 기능 확장, 앱 개발 여부 검토

## 필요 리소스 & 역할

- **PM/서비스 기획**: 1명
- **UX/UI 디자이너**: 1~2명
- **프론트엔드 개발자**: 2명
- **백엔드 개발자**: 1~2명
- **데이터 분석/엔지니어**: 1명
- **마케팅/사업개발**: 1명

## 리스크 & 대응

- **데이터 품질**: 자동 수집 + 관리자 검수 병행, 신뢰도 점수 체계
- **규제 변화**: 법무 자문, 정기적인 제도 업데이트 체크
- **사용자 확보**: 초기 파트너십, 커뮤니티 연계, 리퍼럴 프로그램 설계
- **리소스 한계**: MVP 범위 우선순위 관리, 외주/파트너 협력 고려
